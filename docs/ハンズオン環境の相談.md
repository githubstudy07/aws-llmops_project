# AWS LLMOps プロジェクト：ハンズオン環境の再利用に関する引き継ぎ

## 現在の状況
- プロジェクトルート: `C:\Users\naoji\Documents\projects\aws-llmops_project`
- 現在、一つのハンズオン（LLMOps）が完了、または一区切りの状態です。
- SSD容量（特に `sam_venv` の約350MB）を節約するため、新しいプロジェクトフォルダを作らず、現在のフォルダと `.venv` を使いまわして別のハンズオンを開始したいと考えています。

## 決定した戦略：Gitブランチ運用
- **フォルダを分けず、ブランチで中身を切り替える**: 1つの `.venv` と `.aws-sam` フォルダを共有し、SSD消費を最小限に抑えます。
- **ハンズオンごとにリリースブランチを用意**: `main` (session-01用) だけでなく、`session-02`, `session-03` ...と、ハンズオンごとの親ブランチを作成します。
- **GitHub Actions の個別化**: 各ブランチの `deploy.yml` を書き換え、そのブランチへのPUSHで、それぞれ別のAWSスタック（例: `handson-llmops-v2`）にデプロイされるよう設定します。

## 次に実施したい作業
1. 現在の `main` ブランチの状態をコミットし、きれいに保存する。
2. 新しいハンズオン用のブランチ（例: `session-02`）を作成する。
3. `session-02` ブランチにおいて、GitHub Actions の設定（デプロイ先スタック名やトリガー条件）を書き換える。
4. 混乱を防ぐための「看板ファイル（`CONTEXT.md` など）」を設置する。

## AI（Antigravity）への依頼事項
- 上記「Gitブランチ戦略」に基づき、今の環境を壊さずに新しいハンズオンを開始できる状態へ整理するサポートをお願いします。
