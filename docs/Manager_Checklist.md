# 現場監督用：AWS LLMOps ハンズオン管理ガイド

このドキュメントは、現場監督（ユーザー）が職人（AI）の作業品質を「コードを読まずに」管理・監督するためのチェックリストです。
各機能ごとに「前（Input）」「本番号」「後（Output）」の行を設け、プロセス全体の繋がりをチェックできるようにしています。

## 📋 運用ルール
1. **フェーズ開始前**: AIにそのフェーズの「機能ゴール」と「前後の繋がり」を確認させる。
2. **フェーズ完了時**: AIにこのリストを更新させ、前・本・後の繋がりに対する「証跡（エビデンス）」を提示させる。
3. **承認**: 監督が証拠を確認し、納得した時点で `✅` を記録して次へ進む。
