## 4. テスト結果詳細

### 機能ID: F-004 （API認証機能）

#### シナリオ #4: APIキーなしでリクエストした場合、403エラーが返却されること

| 項目 | 内容 |
|---|---|
| **種別** | 異常系 |
| **優先度** | High |
| **結果** | Pass |
| **実施日** | 2026/04/09 |

**操作と結果:**

| ステップ | 操作内容 | 期待結果 | 実際の結果 | 判定 |
|---|---|---|---|---|
| 1 | `curl.exe`でAPIキーを除外してPOSTリクエストを送信 | ステータスコード 403 が返ること | ステータスコード 403 | ✅ |
| 2 | レスポンスボディを確認 | "Forbidden" メッセージ | `{"message":"Forbidden"}` | ✅ |

**エビデンス:**
```bash
HTTP/1.1 403 Forbidden
x-amzn-ErrorType: ForbiddenException
{"message":"Forbidden"}
```

**判定根拠:** API Gatewayにより期待通り認証なしのリクエストが拒絶された。

### 機能ID: F-001 （AI対話機能）

#### シナリオ #1: 正常なメッセージを送信した場合、AIからの回答が返却されること

| 項目 | 内容 |
|---|---|
| **種別** | 正常系 |
| **優先度** | High |
| **結果** | Fail |
| **実施日** | 2026/04/09 |

**操作と結果:**

| ステップ | 操作内容 | 期待結果 | 実際の結果 | 判定 |
|---|---|---|---|---|
| 1 | AWSから取得した有効なAPIキー（vKi3...）を使用しリクエスト | ステータスコード 200 | ステータスコード 403 | ❌ |
| 2 | レスポンス内容の確認 | AIの回答が含まれること | `{"message":"Forbidden"}` | ❌ |

**エビデンス:**
```json
{
    "id": "jjax2f5v4l",
    "value": "vKi3EE5VUn76kdNFioDvtaoLtmuPKBz5FaesXwC9",
    "name": "LangGraphAgentKey"
}
--> HTTP/1.1 403 Forbidden
```

**判定根拠:** 有効であるはずのAPIキーがAPI Gatewayで拒絶されている。使用プランまたはステージとの紐付けに不備がある可能性が高い。
