**Q1.** 〔カテゴリ: 概念理解〕
現場監督としてプロジェクトのコスト最適化を検討しています。Amazon API Gateway を選定する際、リクエストバリデーションやキャッシュ機能が不要で、低コストかつ低レイテンシを最優先する場合、どの API タイプを選択するのが最も適切でしょうか？

A. REST API
B. HTTP API
C. WebSocket API
D. プライベート API

<details>
<summary>解答・解説を表示</summary>

**正解: B**
**なぜ正解か:** HTTP API は REST API の主要機能（バリデーションやキャッシュ）を削ぎ落とした軽量版であり、コストが REST API の約 70% 安く、低レイテンシという特徴があります。
**不正解の理由:**
- A: REST API は高機能ですが、HTTP API よりもコストが高くなります。
- C: WebSocket API は双方向リアルタイム通信用であり、一般的な Web 要件の低コスト化の第一選択ではありません。
- D: プライベート API は VPC 内限定の公開設定を指すもので、コストメリットを直接目的としたタイプ選択ではありません。

A. ToolNode

<summary>解答・解説を表示</summary>

- A: ToolNode はツールの実行を自動化するためのノードであり、保存機能は持ちません。
- B: TypedDict は状態の型定義（スキーマ）であり、保存の実装そのものではありません。
- D: recursion_limit は無限ループを防ぐためのステップ上限設定です。

- D: タイムアウト設定は接続不可によるエラーを遅らせるだけで、原因の解決になりません。
