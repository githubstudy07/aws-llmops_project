# 調査中：Langfuse プロンプト取得 404 (Prompt not found) エラー

## 1. 現状と到達点
本日のセッションにおいて、以下の「疎通基盤」は完全に確立されました。
- **クライアント初期化**: 成功（`init_err: None`, `has_client: True`）
- **Langfuse Cloud との通信**: 成功（Langfuse サーバーから HTTP 404 レスポンスが返っているため、導通はしている）

## 2. 発生している問題
`main-chatbot-prompt` という名称のプロンプトを `production` ラベルで取得しようとすると、以下のエラーが発生します。
`status_code: 404, body: {'message': 'Prompt not found'}`

## 3. 仮説と検証結果

| 仮説 | 内容 | 検証結果 |
| :--- | :--- | :--- |
| **A. SDK初期化不備** | `get_client` の引数エラー | **解消済み**。`Langfuse` クラスの使用で解決。 |
| **B. ラベル名不一致** | `Production` か `production` か | **解消済み**。両方試したが共に 404。 |
| **C. リージョン不一致** | Host(US) と Project(US) の食い違い | **解消済み**。画像により双方 `us.cloud.langfuse.com` で一致していることを確認。 |
| **D. プロンプト名齟齬** | UI名と内部名称が異なる | **要検証**。ハイフンの種類や不可視文字、またはプロジェクト間での名称不可視の可能性。 |
| **E. ラベル紐付け不備** | Production ラベルの適用状態 | **要検証**。タグではなく「環境ラベル」としての適用が SDK から正しく見えているか。 |

## 4. 次回の具体的アクション
1. **ラベルなし取得のテスト**: 
   - `label=None` にコードを変更し、最新版が取得できるか試します。これで 404 なら「名称」自体の問題です。
2. **新規プロンプト作成テスト**: 
   - 別の極めて単純な名前（例: `test`）でプロンプトを作成し、それが取得できるかを確認します。

## 5. 備考
現在、アプリ側には「詳細な診断情報」をレスポンスに含めるデバッグコードが入ったままになっています。一度プロンプトの反映（ズラ口調）に成功した後は、速やかにこれらを削除し、セキュアな状態に戻す必要があります。
