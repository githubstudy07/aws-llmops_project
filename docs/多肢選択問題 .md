**Q1.** 〔カテゴリ: 概念理解〕

A. LLM（大規模言語モデル）の推論実行そのものを行うエンジン
B. AIエージェントの「状態（State）」や「記憶」を管理し、多段階の処理フローを制御する仕組み
C. AWSリソースを自動デプロイするためのコマンドラインツール
D. ユーザーとのフロントエンドチャット画面を提供するためのライブラリ

<details>
<summary>解答・解説を表示</summary>

**不正解の理由:**
- A: 推論そのものは Amazon Bedrock が担当しています。
- C: デプロイは AWS SAM が担当しています。
- D: 本プロジェクトにはフロントエンド画面のライブラリは含まれておらず、APIとしての提供が主です。
**関連知識:** ステートフル（状態を保持する）な対話。

**Q2.** 〔カテゴリ: 機能間の関係〕
API Gateway から Lambda 関数を呼び出す際、本プロジェクトで採用されている「AWS_PROXY（Lambda プロキシ統合）」の説明として正しいものはどれか。

A. API Gateway がリクエスト内容を解析せず、そのまま（Rawデータ）Lambda に渡し、Lambda のレスポンスもそのままクライアントに返す方式
B. API Gateway が Lambda の戻り値を自動的に HTML 形式に変換してブラウザに表示する方式
C. Lambda を介さず、API Gateway が直接 DynamoDB にデータを書き込む方式
D. セキュリティを確保するために、特定の IP アドレス以外からのアクセスをすべて遮断するプロキシサーバーを立てる方式

<details>
<summary>解答・解説を表示</summary>

**正解: A**
**なぜ正解か:** `template.yaml` で `Type: AWS_PROXY` が指定されており、これは API Gateway が HTTP リクエスト全体（ヘッダー、ボディ等）を `event` オブジェクトとして Lambda に丸投げし、Lambda 側で解析・レスポンス構築を行う柔軟な方式です。
**不正解の理由:**
- B: API Gateway は自動変換を行わず、Lambda 側で `json.dumps` 等を用いてステータスコード等を含む JSON を構築する必要があります。
- C: 本構成では Lambda がビジネスロジックの中核となっており、直接連携はしていません。
- D: AWS_PROXY はデータの渡し方の設定であり、IP 制限の仕組みのことではありません。
**関連知識:** イベント駆動アーキテクチャ。
**該当ファイル:** `template.yaml` (Line 79)

</details>

Lambda 関数内で、Langfuse（モニタリングツール）の API キーなどの機密情報を取得するために使用されているサービスはどれか。

A. Amazon S3（オブジェクトストレージ）
B. AWS Secrets Manager（高機能な秘匿情報管理）
C. AWS SSM Parameter Store（パラメータ管理）
D. AWS Lambda のコード内に直接文字列として記述している

<details>
<summary>解答・解説を表示</summary>

- A: S3 は主に大容量ファイルの保存用であり、設定情報の動的取得には一般的ではありません。
- B: 本プロジェクトではよりシンプルでコスト効率の良い SSM Parameter Store が選択されています。
- D: 進捗ログにもある通り、セキュリティ上の理由からハードコーディング（生書き）は厳禁されています。
**関連知識:** 環境変数の外部管理。

**Q4.** 〔カテゴリ: トラブルシュート〕
LLM（Amazon Bedrock）を呼び出す Lambda 関数のタイムアウト設定（Timeout）を 3秒 ではなく 60秒 などの長めに設定すべき理由として、最も適切なものはどれか。

A. Lambda 関数は起動するたびに課金されるため、1回の実行時間を長くしたほうが安くなるから
B. ネットワークエラーが発生した際に、自動で何度もリトライを繰り返す必要があるから
C. LLM による回答生成には数秒〜数十秒かかることがあり、短いタイムアウトでは処理が完了する前に強制終了してしまうから
D. セキュリティスキャンを毎回実行するのに時間がかかるから

<details>
<summary>解答・解説を表示</summary>

**正解: C**
**なぜ正解か:** LLM（特に複雑な指示や長文生成）は、従来のデータベース処理に比べて応答に時間がかかる性質があるため、サーバーレス関数のタイムアウトを長めに設定するのが一般的です。
**不正解の理由:**
- A: Lambda は実行時間に比例して課金されるため、長く設定しても安くはなりません。
- B: リトライは重要ですが、そもそも1回の推論待ち時間が長いためです。
- D: 実行時に毎回セキュリティスキャンを行うような実装は一般的ではありません。
**関連知識:** レイテンシ（応答遅延）。
**該当ファイル:** `template.yaml` (Line 17)

</details>

**Q5.** 〔カテゴリ: 概念理解〕
本プロジェクトで使用されている「Amazon Bedrock」の説明として、正しいものはどれか。

A. 自社専用のサーバーを用意して、独自の AI モデルをゼロから学習させるためのサービス
B. 複数の AI モデル（Nova, Claude 等）を API 経由で簡単に利用できる、サーバーレスなフルマネージドサービス
C. チャットの履歴を永続的に保存するための Key-Value 型データベース

<details>
<summary>解答・解説を表示</summary>

**不正解の理由:**
- A: モデルの学習ではなく、既存の推論済みモデル（Nova Micro 等）を利用しています。
- C: それは DynamoDB の役割です。
- D: それは IDE や SageMaker Studio などの役割です。
**関連知識:** 基盤モデル（Foundation Models）。

**Q7.** 〔カテゴリ: トラブルシュート〕
Lambda 関数内で Langfuse の `CallbackHandler` を使用してトレーシングを行う際、処理の最後に `handler.flush()` を明示的に呼び出す必要がある理由は何か。

A. 送信中のデータを暗号化して保護するため
B. Lambda は処理が終わると実行環境がすぐに凍結（フリーズ）されるため、バッファに残っているログデータを確実に送信し切る必要があるから
C. DynamoDB に保存されているチャット履歴を削除するため
D. 次のユーザーのリクエストを受け付けられるようにメモリを解放するため

<details>
<summary>解答・解説を表示</summary>

**正解: B**
**なぜ正解か:** Lambda は非同期でログを送る仕組みがある場合、関数終了後に背景で送ろうとしても環境が停止してしまい、ログが消失することがあります。そのため明示的に「今すぐ全部送れ（flush）」と命じる必要があります。
**不正解의 이유:**
- A: flush は「溜まっているものを出す」操作であり、暗号化の有無とは関係ありません。
- C: 履歴の削除は行いません。
- D: メモリ解放は Lambda 環境の破棄時などに自動で行われます。
**関連知識:** サーバーレスのライフサイクル。
**該当ファイル:** `src/app.py` (Line 210)

</details>

**Q8.** 〔カテゴリ: 概念理解〕
プロジェクトのルートディレクトリにある `template.yaml` が準拠している「AWS SAM (Serverless Application Model)」の主なメリットはどれか。

A. 複雑なサーバー構成（OSの設定など）を細かくカスタマイズできること
B. Lambda や API Gateway などのサーバーレスリソースを、簡潔な記述で定義・展開（デプロイ）できること
C. Python 以外の言語での開発を禁止してセキュリティを高めること
D. AWS 以外のクラウド（Azure や Google Cloud）にもそのままデプロイできること

<details>
<summary>解答・解説を表示</summary>

**正解: B**

**不正解の理由:**
- A: SAM（およびサーバーレス）は OS 管理から解放されることがメリットであり、カスタマイズ性は低くなります。
- C: 多言語に対応しており、制限するものではありません。
- D: AWS 専用のツールであり、他社クラウドには対応していません。
**関連知識:** IaC (Infrastructure as Code)。
**該当ファイル:** `template.yaml` (Line 2)

</details>

**Q9.** 〔カテゴリ: 設定・判断〕
IAM ポリシーで `bedrock:InvokeModel` 権限を Lambda 関数に付与している理由として、正しいものはどれか。

A. Lambda が S3 のバケットを作成できるようにするため
B. Lambda がインターネット経由で Google 検索を行えるようにするため
C. Lambda から Amazon Bedrock の AI モデルに対して「推論の実行（呼び出し）」を許可するため
D. API Gateway から Lambda を呼び出す許可を与えるため

<details>
<summary>解答・解説を表示</summary>

**正解: C**
**なぜ正解か:** AWS では「最小権限の原則」に基づき、各サービスへの操作を IAM で明示的に許可する必要があります。`InvokeModel` はモデルを使用するために必須の権限です。
**不正解の理由:**
- A: S3 の権限は `s3:CreateBucket` など別の権限が必要です。
- B: インターネットアクセスはネットワーク設定（VPC/NAT等）の問題であり、IAM ポリシーではありません。
- D: それは Lambda 側の「リソースベースポリシー (`AWS::Lambda::Permission`)」で行います。
**関連知識:** 最小権限の原則 (Least Privilege)。
**該当ファイル:** `template.yaml` (Line 41)

</details>

**Q10.** 〔カテゴリ: 機能間の関係〕
LangGraph が DynamoDB を「チェックポインター（Checkpointer）」として使用している目的は何か。

A. LLM の回答速度を高速化するためのキャッシュとして利用するため
B. 会話の途中の状態を保存しておき、次回のユーザー入力時に「何の話をしていたか」を継続できるようにするため
C. API キーの利用制限（クォータ）を管理するため
D. Lambda 関数のソースコードをバックアップするため

<details>
<summary>解答・解説を表示</summary>

**正解: B**
**なぜ正解か:** 短期間で終了する Lambda（ステートレス）において、会話の文脈を維持するために、外部データベースである DynamoDB に現在のグラフの状態を書き出し、復元する仕組み（チェックポイント）が必要です。
**不正解の理由:**
- A: キャッシュとして使うことも可能ですが、本プロジェクトの主な目的は「継続性」です。
- C: クォータ管理は API Gateway の Usage Plan などの役割です。
- D: コードの管理は GitHub が担当しています。
**関連知識:** ステート（状態）の永続化。
**該当ファイル:** `src/app.py` (Line 149-153)

</details>
