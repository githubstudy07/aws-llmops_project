# テストケース一覧

## 1. テスト観点の分類体系

今回の要件に基づき、以下の分類軸でテスト観点を構成しました。

| 分類軸 | 内容 |
|---|---|
| **機能テスト** | AI対話、履歴保存、入力バリデーションの正常・異常系 |
| **データテスト** | JSON構造、特殊文字、空文字、メッセージの長さ |
| **権限・認証テスト** | API Keyの有無、無効なキーでのアクセス制限 |
| **外部連携テスト** | Bedrock呼び出し、DynamoDBへの書き込み・読み込み |
| **非機能テスト（性能）** | タイムアウト設定、レスポンスの遅延対応 |
| **非機能テスト（セキュリティ）** | HTTPS通信、最小権限（IAM）、CORS設定 |
| **非機能テスト（可用性）** | Bedrock一時障害時のリトライ挙動 |
